BUTTON TYPES BREAKDOWN
================================================================================

1. PRIMARY ACTIONS (34% of buttons)
   - Purpose: Main CTAs in modals, forms
   - Pattern: Solid color + darker hover + white text
   - Files: ExportButton, NewTaskModal, ConfirmDialog
   - Example: "Save Task", "Export", "Confirm"

2. SECONDARY ACTIONS (23% of buttons)
   - Purpose: Less critical (Cancel, Skip, Back)
   - Pattern: Gray background + text variant
   - Files: Modal footers, form cancellation
   - Example: "Cancel", "Skip", "Back"

3. ICON BUTTONS (20% of buttons)
   - Purpose: Single action, no text
   - Pattern: Circle + hover background + scale feedback
   - Files: Modal close, theme toggle, navigation
   - Example: X icon, settings icon, menu toggle

4. COMPACT ACTIONS (15% of buttons)
   - Purpose: Multiple quick actions in groups
   - Pattern: Icon + small text + gray/action color
   - Files: TaskQuickActions, BulkActionsBar
   - Example: Complete, Snooze, Assign, Delete

5. SELECTION TOGGLES (4% of buttons)
   - Purpose: Choose between options
   - Pattern: Border-based + colored when selected
   - Files: ExportButton format selection
   - Example: CSV vs PDF, timeframe selection

6. MENU ITEMS (2% of buttons)
   - Purpose: Dropdown/context menu options
   - Pattern: Full width + left-aligned + hover background
   - Files: Bulk action menus
   - Example: Status options, priority options

7. FORM SUBMISSION (2% of buttons)
   - Purpose: Form submission in modals
   - Pattern: Full width + primary color + loading state
   - Files: NewTaskModal, NewMessageModal
   - Example: "Save Changes", "Create Item"

ADVANCED FEATURES (Lower Priority):

MOBILE OPTIMIZATION PATTERNS
================================================================================

Touch Target Sizing:
  - Icon buttons: 48x48px (mobile), 40x40px (desktop)
  - Min padding: py-1.5 on mobile, py-1 on desktop
  - Form buttons: py-3 (mobile), py-2.5 (desktop)

Responsive Breakpoints:
  - Mobile: Full-width buttons (w-full)
  - Desktop: Auto-width with max constraints (sm:w-auto)
  - Icon sizes scale: w-5 h-5 (mobile) â†’ w-4 h-4 (desktop)

Touch Feedback:
  - active:scale-95 on all pressable buttons
  - Transition duration: 100ms for quick feedback
  - No hover effects on mobile (touch-based instead)

================================================================================
ACCESSIBILITY COMPLIANCE
================================================================================

Keyboard Navigation:
  - focus:outline-none + focus:ring-4 on all buttons
  - Proper focus rings: sm:focus:ring-2 (desktop), focus:ring-4 (mobile)
  - Ring offset for visual clarity: focus:ring-offset-2

Screen Reader Support:
  - aria-label on icon-only buttons
  - Semantic HTML: <button>, <form>, type="submit"
  - Descriptive text for button actions

Color Contrast:
  - WCAG AA compliant across all color combinations
  - Text: White on dark backgrounds, dark on light
  - Icons: Sufficient color contrast

Motion Preferences:
  - @media (prefers-reduced-motion: reduce) support in globals.css
  - Animations can be safely disabled for accessibility
