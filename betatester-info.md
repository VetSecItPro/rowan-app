## Feedback Categories

### Bug Severity Levels
1. **Critical**: App crashes, data loss, security issues
2. **High**: Feature doesn't work, blocks main workflows
3. **Medium**: Minor functionality issues, UI glitches
4. **Low**: Cosmetic issues, nice-to-have improvements

### Feature Priority Levels
1. **Must Have**: Essential for launch
2. **Should Have**: Important for user experience
3. **Could Have**: Nice additions for future versions
4. **Won't Have**: Not planned for current version

## Testing Tips

### Best Practices
1. **Test on Multiple Devices**
   - Desktop (Chrome, Firefox, Safari)
   - Mobile (iOS Safari, Android Chrome)
   - Different screen sizes

2. **Test Different Scenarios**
   - Empty states (no tasks, no recipes, etc.)
   - Full states (lots of data)
   - Edge cases (very long text, special characters)

3. **Document Everything**
   - Take screenshots of issues
   - Note the exact steps you took
   - Include device and browser information

### Common Testing Scenarios
1. **New User Experience**
   - Sign up flow
   - Onboarding process
   - First-time feature usage

2. **Power User Workflows**
   - Managing multiple spaces
   - Complex meal planning
   - Advanced budget tracking

3. **Collaboration Testing**
   - Invite family members
   - Share lists and tasks
   - Multi-user editing
